# v0.1.7 Release Notes

## English

v0.1.7 makes GyShell more "ship-ready": smoother execution flow, clearer session state, and stronger multi-surface consistency.

### Highlights You Will Notice

- **Smoother chat-first workflow**: cleaner TUI rhythm, clearer run-state feedback, and better long-session readability.
- **More reliable session behavior**: run/stop/ready transitions are more predictable, especially when sessions are busy.
- **Profile lock during active runs**: model profile stays pinned while a session is running, which reduces unexpected behavior drift.
- **Better multi-client consistency**: desktop, TUI, and mobile-web now stay more aligned on session status and terminal tabs.
- **OpenClawd-style remote control**: keep the runtime core on your own computer and control active work from anywhere through conversation.

### Mobile-Web Is Now More Practical

- **Session Browser** with search and run indicators to jump between conversations faster.
- **Message Detail Sheet** to inspect the detailed events behind one response turn.
- **Tools Panel** to manage both:
  - MCP server enable/disable and status
  - Built-in tool enable/disable
- **Remote steering flow**: mobile-web is now practical for monitoring and steering live sessions outside your desk setup.

### Desktop-Bundled `gyll` CLI

- `gyll` is bundled with desktop runtime setup.
- If `--url` is omitted, it auto-connects to local desktop backend (`127.0.0.1:17888` by default).
- Common commands:
  - `gyll`
  - `gyll "message"`
  - `gyll run "message"`
  - `gyll hook "message"`
  - `gyll --url ip:port ...`
  - `gyll --sessionid <id>`

Mode semantics:

- `gyll`: interactive TUI.
- `gyll "message"`: create session, send immediately, then enter TUI.
- `gyll run "message"`: create session, stream output directly, no TUI.
- `gyll hook "message"`: create session, send once, then exit.

### Runtime and Connectivity Improvements

- **WebSocket gateway policy control** is clearer (`disabled` / `localhost` / `internet`).
- **Gateway lifecycle synchronization** is more stable across transports.
- **MCP stdio startup is more robust**:
  - better PATH merging for command resolution
  - safer CWD fallback strategy
- **Parallel terminal control**: agent can coordinate multiple local/SSH terminal tabs in one task workflow.

## 中文

v0.1.7 让 GyShell 更接近“可交付状态”：执行链路更顺、会话状态更清晰、多端一致性更强。

### 你会直接感知到的升级

- **chat-first 体验更顺**：TUI 节奏更清爽，运行状态反馈更清晰，长对话更好读。
- **会话行为更稳定**：运行/停止/就绪的状态切换更可预期，尤其是在会话繁忙时。
- **运行中 Profile Lock**：会话执行期间锁定模型配置，减少中途配置变化带来的行为漂移。
- **多端一致性更好**：桌面端、TUI、Mobile Web 对会话状态与终端标签的同步更一致。
- **OpenClawd 风格远程控制**：核心运行在你自己的电脑上，你可以在任何地方通过对话持续控制任务。

### Mobile Web 变得真正可用

- **会话浏览器**：支持搜索和运行状态标记，切换会话更快。
- **消息详情面板**：可查看单轮响应背后的细粒度事件。
- **Tools 管理面板**：支持
  - MCP 服务器启用/停用与状态查看
  - 内置工具启用/停用
- **远程接管更自然**：离开电脑后也能通过手机端持续查看并引导活跃会话。

### 桌面版内置 `gyll` CLI

- `gyll` 已集成在桌面运行时安装链路中。
- 不传 `--url` 时，默认自动尝试连接本机桌面后端（`127.0.0.1:17888`）。
- 常见命令：
  - `gyll`
  - `gyll "消息"`
  - `gyll run "消息"`
  - `gyll hook "消息"`
  - `gyll --url ip:port ...`
  - `gyll --sessionid <id>`

模式语义：

- `gyll`：进入交互式 TUI。
- `gyll "消息"`：新建会话并立即发送，然后进入 TUI。
- `gyll run "消息"`：新建会话并直接在终端流式输出，不进入 TUI。
- `gyll hook "消息"`：新建会话，发送一次后立即退出。

### 运行时与连接能力改进

- **WebSocket 网关策略更清晰**（`disabled` / `localhost` / `internet`）。
- **网关生命周期同步更稳定**，跨传输行为更一致。
- **MCP stdio 启动可靠性提升**：
  - PATH 合并更稳，减少命令找不到的问题
  - CWD 回退策略更安全
- **并行终端控制增强**：Agent 可在单个任务中同时协调多个本地/SSH terminal tab。
