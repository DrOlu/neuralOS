## English

v1.1.0 turns GyShell into a more complete execution workspace: multimodal input, persistent memory, smarter long-history handling, a fully flexible desktop layout, and stronger runtime recovery after restarts.

Usage guides:
- Mobile-Web: [`docs/mobile-web-usage.md`](../docs/mobile-web-usage.md)
- TUI (`gyll`): [`docs/tui-usage.md`](../docs/tui-usage.md)
- GyBackend: [`docs/gybackend-usage.md`](../docs/gybackend-usage.md)

### Highlights You Will Notice

- **Multimodal chat input**: attach images from desktop and mobile-web (paste/drag/drop), preview/remove before send, and let supported models reason over image + text in one request.
- **Persistent global memory (`memory.md`)**: keep durable project rules and facts that can be injected into system prompts; manageable in desktop settings and readable from mobile-web.
- **Smarter long-session compaction**: old context is compacted with model-generated summaries instead of blunt truncation, keeping recent rounds intact and reducing context-window pressure.
- **Compaction visibility across surfaces**: desktop, mobile-web, and TUI now show dedicated compaction activity banners so history maintenance is explicit, not invisible.
- **Terminal resilience after restart**: terminal tabs are persisted and restored automatically; offset-based buffer delta sync provides lossless output catch-up when the terminal view remounts/reconnects within the same backend runtime.

### Desktop Workspace Upgrade

- **Splitter layout for all panels**: freely split chat/terminal panels in all directions, resize, and persist the workspace tree.
- **Panel-type rail**: one-click create/focus for chat and terminal workspaces with live tab counts.
- **Advanced tab operations**: drag-and-drop reorder/move, split-to-direction, close-other-tabs, and drop-to-trash interactions for faster workspace management.

### Mobile-Web Improvements

- **Swipe-to-delete sessions** in the session browser for faster cleanup on phone.
- **Image composer polish**: simpler image-chip controls and better paste extraction behavior.
- **Settings parity for memory visibility**: mobile can inspect memory status/path/content in read-only mode.

### Runtime and Reliability Hardening

- Prompt assembly is refactored to avoid duplicate system context and reduce local-model noise.
- System-info injection now works with both plain text and multimodal message content.
- SSH bootstrap flow is hardened with better ready-marker handling and safer initialization retries.
- Gateway/IPC adds terminal buffer delta RPC to support robust renderer-side output sync.
- Extreme test coverage is expanded for layout restoration and terminal persistence paths.

## 中文

v1.1.0 让 GyShell 更像一个完整的执行工作台：支持多模态输入、全局记忆、长历史智能压缩、桌面端自由布局，以及重启后的终端 tab 恢复与运行期输出补齐。

使用教程：
- Mobile-Web：[`docs/mobile-web-usage.md`](../docs/mobile-web-usage.md)
- TUI（`gyll`）：[`docs/tui-usage.md`](../docs/tui-usage.md)
- GyBackend：[`docs/gybackend-usage.md`](../docs/gybackend-usage.md)

### 你会直接感知到的升级

- **多模态对话输入**：桌面端与 Mobile-Web 支持图片附件（粘贴/拖拽/选择），发送前可预览/移除，支持的模型可在同一轮同时理解文字与图片。
- **全局持久记忆（`memory.md`）**：可沉淀长期规则和项目信息，并按设置注入系统提示；桌面端可编辑，移动端可只读查看。
- **长会话智能压缩**：不再是粗暴裁剪，旧上下文会被模型总结压缩，尽量保留最近轮次，降低上下文窗口压力。
- **压缩过程可视化**：桌面端、Mobile-Web、TUI 都新增了 compaction 活动展示，让历史维护过程可见、可感知。
- **终端重启恢复能力**：terminal tabs 可持久化并自动恢复；基于 offset 的增量同步用于同一后端运行期内终端视图重挂载/重连时的输出无损补齐。

### 桌面工作区升级

- **全面板分割布局**：聊天与终端面板可任意方向分割、拖拽调节尺寸，并持久化整个布局树。
- **面板类型侧栏（Rail）**：可一键创建/聚焦聊天与终端工作区，并显示实时 tab 数量。
- **更强的标签管理**：支持拖拽重排/移动、按方向拆分、关闭其他标签、拖入回收区关闭等高频操作。

### Mobile-Web 改进

- **会话列表支持左滑删除**，手机端清理会话更高效。
- **图片输入区细节优化**：图片 chip 与清空交互更直观，粘贴提取逻辑更稳定。
- **Memory 设置可见性增强**：移动端可在设置中查看 memory 状态、路径与内容（只读）。

### 运行时与稳定性增强

- 提示词装配链路重构，减少重复系统上下文和对本地模型的噪声干扰。
- 系统信息注入兼容纯文本与多模态消息内容。
- SSH 初始化链路增强：ready marker 处理更稳、重试注入更安全。
- 网关/IPC 新增终端缓冲区增量 RPC，前端输出同步更可靠。
- 布局恢复与终端持久化路径补充了更完整的 extreme 测试覆盖。
