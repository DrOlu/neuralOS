## English

v1.1.0 turns GyShell into a more complete execution workspace: multimodal input, persistent memory, smarter long-history handling, a fully flexible desktop layout with integrated file management, stronger runtime recovery after restarts, and tighter agent safety controls.

Usage guides:
- Mobile-Web: [`docs/mobile-web-usage.md`](../docs/mobile-web-usage.md)
- TUI (`gyll`): [`docs/tui-usage.md`](../docs/tui-usage.md)
- GyBackend: [`docs/gybackend-usage.md`](../docs/gybackend-usage.md)

### Highlights You Will Notice

- **Multimodal chat input**: attach images from desktop and mobile-web (paste/drag/drop), preview/remove before send, and let supported models reason over image + text in one request.
- **Persistent global memory (`memory.md`)**: keep durable project rules and facts that can be injected into system prompts; manageable in desktop settings and readable from mobile-web.
- **Smarter long-session compaction**: old context is compacted with model-generated summaries instead of blunt truncation, keeping recent rounds intact and reducing context-window pressure.
- **Compaction visibility across surfaces**: desktop, mobile-web, and TUI now show dedicated compaction activity banners so history maintenance is explicit, not invisible.
- **Terminal resilience after restart**: terminal tabs are persisted and restored automatically; offset-based buffer delta sync provides lossless output catch-up when the terminal view remounts/reconnects within the same backend runtime.
- **Integrated file browser and editor**: a visual file manager and built-in text editor are now first-class panels in the desktop workspace, with full SFTP support for SSH sessions.

### Desktop Workspace Upgrade

- **Splitter layout for all panels**: freely split chat/terminal/file panels in all directions, resize, and persist the workspace tree.
- **Panel-type rail**: one-click create/focus for chat, terminal, and file workspaces with live tab counts.
- **Advanced tab operations**: drag-and-drop reorder/move, split-to-direction, close-other-tabs, and drop-to-trash interactions for faster workspace management.

### Integrated File Management

- **File browser panel** as a first-class workspace panel alongside terminals and chat:
  - Navigate directories via path bar, back-to-parent button, or direct path entry
  - Create files and directories inline, rename, and delete with confirmation
  - Multi-select with Shift+click; bulk delete supported
  - **Copy/Cut/Paste across terminal tabs** — works across SSH sessions via SFTP
  - Transfer task panel shows real-time progress (bytes, file count, %) with per-task cancellation
  - Automatic conflict detection on paste with overwrite confirmation
  - Inline text preview for common file types; double-click opens file in editor panel
- **Built-in text editor panel** — edit files directly in the workspace without leaving GyShell:
  - Unsaved-changes indicator; save with Ctrl/Cmd+S
  - Context-menu copy/paste with full keyboard navigation
- **Adaptive SFTP transfer tuner**: auto-selects the best chunk size and concurrency profile per endpoint and direction using EWMA throughput tracking — transfers improve as the system learns your server

### Mobile-Web Improvements

- **Swipe-to-delete sessions** in the session browser for faster cleanup on phone.
- **Image composer polish**: simpler image-chip controls and better paste extraction behavior.
- **Settings parity for memory visibility**: mobile can inspect memory status/path/content in read-only mode.

### Agent Safety Controls (Experimental)

Two new per-session double-audit toggles for tighter action oversight, both on by default and configurable in Settings → Experimental:

- **Exec Command Double Audit**: action model runs a parallel audit on each `exec_command` call to determine whether to switch automatically from wait → nowait mode
- **Write Stdin Double Audit**: action model inspects each `write_stdin` call before execution and can block it if the intent looks incorrect (e.g. wrong key sequence sent to an interactive program)

Both require an Action Model configured in the active profile.

### Runtime and Reliability Hardening

- Prompt assembly is refactored to avoid duplicate system context and reduce local-model noise.
- System-info injection now works with both plain text and multimodal message content.
- SSH bootstrap flow is hardened with better ready-marker handling and safer initialization retries.
- SFTP channel initializes automatically on SSH session start, with graceful degradation when unavailable.
- Gateway/IPC adds terminal buffer delta RPC and filesystem operation RPCs to support robust renderer-side sync.
- Extreme test coverage is expanded for layout restoration, terminal persistence, and filesystem transfer paths.
- Fixed a crash when a skill tool call returns an error — the agent no longer tries to parse the success tag from error message content.

## 中文

v1.1.0 让 GyShell 更像一个完整的执行工作台：支持多模态输入、全局记忆、长历史智能压缩、桌面端自由布局与集成文件管理、重启后终端 tab 恢复与运行期输出补齐，以及更细粒度的 Agent 安全控制。

使用教程：
- Mobile-Web：[`docs/mobile-web-usage.md`](../docs/mobile-web-usage.md)
- TUI（`gyll`）：[`docs/tui-usage.md`](../docs/tui-usage.md)
- GyBackend：[`docs/gybackend-usage.md`](../docs/gybackend-usage.md)

### 你会直接感知到的升级

- **多模态对话输入**：桌面端与 Mobile-Web 支持图片附件（粘贴/拖拽/选择），发送前可预览/移除，支持的模型可在同一轮同时理解文字与图片。
- **全局持久记忆（`memory.md`）**：可沉淀长期规则和项目信息，并按设置注入系统提示；桌面端可编辑，移动端可只读查看。
- **长会话智能压缩**：不再是粗暴裁剪，旧上下文会被模型总结压缩，尽量保留最近轮次，降低上下文窗口压力。
- **压缩过程可视化**：桌面端、Mobile-Web、TUI 都新增了 compaction 活动展示，让历史维护过程可见、可感知。
- **终端重启恢复能力**：terminal tabs 可持久化并自动恢复；基于 offset 的增量同步用于同一后端运行期内终端视图重挂载/重连时的输出无损补齐。
- **集成文件管理与内置编辑器**：可视化文件浏览器与文本编辑器作为一等面板直接集成进桌面工作区，SSH 会话通过 SFTP 完整支持。

### 桌面工作区升级

- **全面板分割布局**：聊天、终端、文件面板可任意方向分割、拖拽调节尺寸，并持久化整个布局树。
- **面板类型侧栏（Rail）**：可一键创建/聚焦聊天、终端与文件工作区，并显示实时 tab 数量。
- **更强的标签管理**：支持拖拽重排/移动、按方向拆分、关闭其他标签、拖入回收区关闭等高频操作。

### 集成文件管理

- **文件浏览面板**作为一等面板与终端/聊天并排存在：
  - 通过路径栏、返回上级按钮或直接输入路径进行目录导航
  - 内联创建文件/目录、重命名、带确认弹窗的删除操作
  - Shift+点击多选，支持批量删除
  - **跨 terminal tab 复制/剪切/粘贴** — 通过 SFTP 可跨 SSH 会话传输
  - 传输任务面板展示实时进度（字节数、文件数、百分比），每项任务均可单独取消
  - 粘贴时自动检测命名冲突，并弹出覆盖确认
  - 单击支持常见文件类型内联预览；双击在编辑器面板中打开文件
- **内置文本编辑器面板** — 无需离开 GyShell 即可直接编辑文件：
  - 未保存变更提示；Ctrl/Cmd+S 保存
  - 右键上下文菜单复制/粘贴，完整键盘导航支持
- **SFTP 自适应传输调优器**：根据每个端点和方向（上传/下载）的历史吞吐量自动选择最优分块大小与并发配置 — 越用越快

### Mobile-Web 改进

- **会话列表支持左滑删除**，手机端清理会话更高效。
- **图片输入区细节优化**：图片 chip 与清空交互更直观，粘贴提取逻辑更稳定。
- **Memory 设置可见性增强**：移动端可在设置中查看 memory 状态、路径与内容（只读）。

### Agent 安全控制（实验性）

两个新增的二次审计开关，均默认开启，可在 设置 → 实验性功能 中调整：

- **Exec Command 双重审计**：action model 对每次 `exec_command` 调用并行审计，决定是否自动从 wait → nowait 模式切换
- **Write Stdin 双重审计**：action model 在每次 `write_stdin` 执行前进行检查，若意图看起来有误（例如发送了错误的按键序列）则可拦截

两者均需要在活跃 Profile 中配置 Action Model。

### 运行时与稳定性增强

- 提示词装配链路重构，减少重复系统上下文和对本地模型的噪声干扰。
- 系统信息注入兼容纯文本与多模态消息内容。
- SSH 初始化链路增强：ready marker 处理更稳、重试注入更安全。
- SSH 会话建立时自动初始化 SFTP 通道，不可用时平滑降级并给出提示。
- 网关/IPC 新增终端缓冲区增量 RPC 与文件系统操作 RPC，前端同步更可靠。
- 布局恢复、终端持久化与文件系统传输路径补充了更完整的 extreme 测试覆盖。
- 修复了 skill 工具调用返回错误时的崩溃问题 — agent 不再尝试从错误消息内容中解析成功标记。
