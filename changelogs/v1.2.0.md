# v1.2.0 Release Notes

## English

v1.2.0 makes GyShell much easier to spread across screens and devices: you can tear panels into their own windows, move tabs and full panels across windows, serve the mobile-web companion directly from the desktop app, inspect live resource usage for local and SSH sessions, and run a real packaged desktop build on Linux.

### Highlights You Will Notice

- **Detachable multi-window workspace**
  - detach chat, terminal, file, file editor, and monitor panels into their own sub-windows
  - drag tabs or full panels across windows instead of forcing everything back into one main workspace
  - detached windows now hand off chat/file-editor state more reliably and close cleanly with the main window
- **Adaptive panel tab display modes**
  - choose `Auto`, `Expanded`, or `Select` in `Settings -> Appearance`
  - auto mode switches chat, terminal, filesystem, and monitor panels to a compact selector on narrow headers
  - compact mode still keeps tab switching and close actions within reach when a window gets tight
- **Built-in Mobile Web server in desktop**
  - start a mobile-web HTTP server directly from `Settings -> Gateway`
  - choose auto or manual port selection and copy ready-to-open LAN access links
  - generated links prefill the gateway address for phone/browser clients
- **More practical gateway exposure controls**
  - new `LAN only` mode exposes the gateway to private-network devices without opening it to every source
  - new `Custom IP ranges` mode lets you restrict access to specific CIDR blocks
- **Resource monitor panel**
  - open a first-class monitor panel from the workspace rail
  - view live CPU, memory, disk, network, process, socket, and GPU information when available
  - works with both local terminals and SSH-backed terminals
- **Linux desktop support**
  - desktop packaging now covers Linux `x64` and `arm64`
  - release artifacts include AppImage, deb, pacman, and rpm outputs
  - Linux builds get proper window chrome behavior, taskbar icon handling, and bundled CLI runtime

### Workspace and Reliability Improvements

- Cross-window moves now preserve transferred chat/file-editor state much more reliably, instead of leaving detached windows or moved tabs in half-restored states.
- Detached child windows shut down with the main desktop window instead of lingering in an inconsistent state.
- Panel-heavy layouts can now stay usable on narrow window headers by switching panel tabs between full strips and a compact stacked selector.
- Monitor tabs are tied to terminal capabilities, so local and SSH sessions expose the same monitor entry points as other workspace panels.

### Mobile and Remote Flow

- Desktop can now publish the mobile-web client itself, so phone access no longer depends on separately hosting the frontend first.
- The desktop settings page now exposes gateway access policy, CIDR allowlists, mobile-web server state, port selection, and copyable access links in one place.
- Remote clients fit the same gateway model as desktop and TUI, reducing setup friction across devices.

## 中文

v1.2.0 让 GyShell 更适合跨屏幕、跨设备协作：你可以把面板拆到独立窗口、在窗口之间拖动标签和整块面板、直接由桌面端托管 Mobile Web、查看本地与 SSH 会话的实时资源占用，并正式使用 Linux 桌面版。

### 你会直接感知到的升级

- **可拆分的多窗口工作区**
  - 聊天、终端、文件、内置编辑器、监控面板都可以独立到子窗口
  - 标签和整块面板都能在窗口之间直接拖动，不再只能回到主窗口里整理
  - 独立窗口在移动聊天/编辑器状态时更稳，主窗口关闭时也会更干净地联动关闭
- **更灵活的面板标签显示模式**
  - 可在 `Settings -> Appearance` 中切换 `自动`、`展开`、`Select`
  - 自动模式会在聊天、终端、文件、监控这些 panel 的 header 过窄时切到紧凑选择器
  - 窄窗口下依然能直接切换和关闭标签，不需要为了省空间牺牲可操作性
- **桌面端内置 Mobile Web 服务**
  - 直接在 `Settings -> Gateway` 中启动 Mobile Web HTTP 服务
  - 支持自动端口或手动端口，并可直接复制可访问的局域网链接
  - 生成的访问链接会自动带上网关地址，手机浏览器打开后无需再手填一遍
- **更实用的网关暴露控制**
  - 新增 `仅局域网` 模式，可以只允许私网设备接入，而不是对所有来源开放
  - 新增 `自定义 IP 范围` 模式，可以用 CIDR 精确限制可连接的地址段
- **资源监控面板**
  - 工作区 Rail 中新增一等面板入口
  - 可查看 CPU、内存、磁盘、网络、进程、套接字，以及可用时的 GPU 信息
  - 本地终端和 SSH 终端都可用
- **Linux 桌面版支持**
  - 正式覆盖 Linux `x64` 与 `arm64`
  - 发布产物包含 AppImage、deb、pacman、rpm
  - Linux 版本补齐了窗口标题栏行为、任务栏图标和内置 CLI 运行时

### 工作区与稳定性改进

- 跨窗口移动聊天或文件编辑器时，状态恢复链路更稳，不容易出现窗口还在但内容没接上的情况。
- 主窗口退出时会级联关闭子窗口，避免留下状态不一致的独立窗口。
- 多 panel 布局在窄窗口下也更容易维持可用，panel tab 现在可以在完整标签栏和紧凑堆叠选择器之间切换。
- 监控面板现在跟终端能力模型统一，本地与 SSH 会话都能通过一致的入口打开。

### 移动端与远程访问体验

- 现在桌面端自己就能托管 Mobile Web，因此手机访问不再依赖你先单独部署前端页面。
- 桌面设置页把网关暴露策略、CIDR 白名单、Mobile Web 服务开关、端口设置和访问链接收拢到了同一个位置。
- 桌面端、TUI、Mobile Web 共用同一套网关语义，跨设备接入成本更低。
