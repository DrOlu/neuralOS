# v1.3.0 Release Notes

## English

v1.3.0 focuses on making long-running work cleaner and easier to control: you can dedicate a separate model to history compaction, draft terminal commands from the current tab context, switch the chat panel into a new Seamless display mode, use a consistent `Ctrl/Cmd+F` find bar across multiple panels, sort and filter files directly inside the workspace browser, and rely on a smarter resource monitor that behaves much better across SSH tabs and Windows targets.

### Highlights You Will Notice

- **Dedicated Compaction Model per profile**
  - assign a separate model just for long-context history summarization inside model profiles
  - keep your main Global, Thinking, and Action models focused on live work while older history is compressed by a different model
  - multimodal history now degrades more predictably when a profile mixes image-capable and text-only models
- **Terminal Command Draft**
  - open a command-draft popover inside the current terminal tab with a configurable shortcut
  - describe what you want to do, and GyShell uses recent visible terminal output plus the selected profile to draft a shell command or snippet
  - the generated command is pasted back into the current terminal instead of being executed automatically, so you stay in control before pressing Enter
- **Seamless chat display mode**
  - switch between `Classic` and `Seamless` in Settings
  - consecutive commands, tool calls, file edits, and sub-tools collapse into one compact progress banner
  - approvals and alerts float above the input area instead of fragmenting the transcript
- **Unified panel-local search**
  - press `Ctrl/Cmd+F` in terminal, the current chat, the file browser, or the file editor
  - move through matches with one consistent find bar instead of learning different per-panel search flows
  - file browser search filters and highlights matching names, while chat search jumps directly between matching messages
- **File browser quality-of-life upgrade**
  - sort entries by name, modified time, size, or type directly in the file panel
  - toggle hidden files without leaving the current directory
  - refresh the built-in text editor without closing and reopening the file
- **Smarter remote monitor sessions**
  - monitor collection is shared across tabs that point at the same local machine or SSH target, reducing duplicate polling
  - if the original source tab exits, monitoring can fail over to another retained tab on the same target
  - remote Windows monitoring now launches through a more reliable PowerShell stdin path and can recover from prompt noise
- **Richer GPU visibility and denser monitor UI**
  - Linux and Windows snapshots now surface VRAM usage percentage; Windows can also expose shared GPU memory when available
  - GPU data gets its own card in compact layouts instead of being mixed into storage
  - the monitor panel now uses a denser CLI-inspired presentation that fits more signal into the same space

### Workflow Polish

- Opening `Settings` no longer blurs and disables the main workspace behind it, which makes side-by-side tuning easier while you keep watching live tabs.

### What This Means in Practice

- Long sessions are easier to tune for cost and speed.
- When you already know the outcome you want, getting the right shell command from the current tab context is much faster.
- Agent runs are easier to scan when tool chatter gets heavy.
- Finding old output, messages, or files is much faster inside the panel you are already using.
- Filesystem work needs fewer context switches.
- Monitoring is more trustworthy in multi-tab remote workflows.

## 中文

v1.3.0 重点在于让长流程任务更干净、更可控：你现在可以为历史压缩单独指定模型，基于当前 terminal 上下文生成命令草稿，把对话面板切到新的无感显示模式，在多个 panel 中使用统一的 `Ctrl/Cmd+F` 搜索条，直接在工作区文件面板里排序和筛选文件，并获得在多 SSH 标签页与 Windows 远程环境下都更稳的资源监控体验。

### 你会直接感知到的升级

- **每个 Profile 都能单独指定压缩模型**
  - 可以在模型配置里为长上下文历史压缩单独指定一个模型
  - 让 Global、Thinking、Action 模型专注当前任务，把旧历史总结交给另一个更合适的模型处理
  - 当一个 Profile 同时混用了支持图片和不支持图片的模型时，历史降级行为会更可预期
- **终端命令草稿**
  - 可通过可配置快捷键在当前 terminal tab 内直接打开命令草稿悬浮窗
  - 描述你想做的事后，GyShell 会结合当前标签页最近可见输出和所选 Profile 生成对应的 shell 命令或片段
  - 生成结果会先粘回当前终端，不会自动执行，仍由你自己确认后再回车
- **无感对话显示模式**
  - 可在设置中切换 `传统模式` 与 `无感模式`
  - 连续的命令、工具调用、文件编辑、子工具步骤会合并成一个紧凑进度横幅
  - 审批请求和警告会浮动到输入区上方，不再把主对话流切得很碎
- **统一的 panel 内搜索**
  - 在终端、当前对话、文件浏览器、文件编辑器里都可以直接按 `Ctrl/Cmd+F`
  - 用同一种查找条在不同 panel 里前后跳转，不需要再记不同的搜索入口
  - 文件面板会直接筛掉不匹配的名称并高亮命中项，对话搜索则会在匹配消息之间跳转
- **文件面板体验升级**
  - 可直接按名称、修改时间、大小、类型排序
  - 可一键切换是否显示隐藏文件，而不需要离开当前目录
  - 内置编辑器支持直接刷新，不必关闭后再重新打开文件
- **更聪明的远程监控会话**
  - 指向同一台本机或同一 SSH 目标的多个标签页会共享监控采样，减少重复轮询
  - 如果最初承担采样的标签页退出，监控可以自动切到同目标下仍保留的其他标签页
  - 远程 Windows 监控改为更稳的 PowerShell stdin 启动方式，并能容忍提示符噪声
- **更丰富的 GPU 可视化与更紧凑的监控布局**
  - Linux 和 Windows 现在都会展示更完整的显存占用比例；Windows 在可用时还会展示共享显存
  - 紧凑布局下，GPU 信息拥有独立卡片，不再和存储信息混在一起
  - 整个监控面板改成更接近 CLI 工具的高密度风格，在同样空间里塞进更多有效信息

### 工作流细节优化

- 打开 `Settings` 时，主工作区不再被模糊并禁用，你可以一边调参数一边对照当前会话和面板状态。

### 这次升级带来的实际体验

- 长会话更容易在成本、速度和效果之间做取舍。
- 当你已经知道要达成什么结果时，基于当前终端上下文补出正确命令会更快。
- Agent 执行步骤很多时，对话阅读负担更低。
- 找旧输出、旧消息、旧文件时，不用离开当前 panel 就能更快定位。
- 文件操作链路更顺手，来回切换更少。
- 多标签远程监控场景下，数据更稳定也更可信。
